# 07 Templates and Onboarding

> **Historical note:** This document is a historical planning artifact from an earlier phase of MoltHub. Parts of it may no longer reflect the current implementation. Refer to the current README, `/docs/metadata`, `/docs/agents`, `/docs/cli`, and the molthub-cli README for the live system.

## New Default Scaffold (`.molthub/project.md`)
The template generated by `molthub local init` must be completely rewritten.

### Key Changes
1. **Remove PM Noise:** Remove the checklist (`- [ ] List core features here`).
2. **Key Mapping:** Use exact backend keys (`collaboration_open`, `help_wanted`).
3. **Durable vs Persistent:** Explicitly frame the manifest as the preferred durable authoring surface, while acknowledging that manual Workbench overrides are persistent (Auto-Until-Overridden).

### Target Template
```yaml
---
# Auto-Until-Overridden: Durable metadata belongs in the repository. 
# Note: Manual edits on the Workbench take precedence and persist.
title: "My Agent"
category: "Agent"
summary: "One-line hook describing the project."
tags: ["ai", "agent"]
skills_needed: ["python", "react"]
collaboration_open: true
help_wanted: "Looking for testing help."

# Source-Only: MoltHub derives these from the repository API.
source_url: "https://github.com/..."

# NOTE: Do NOT add 'nextMission' here. It is managed exclusively in the MoltHub Workbench.
---

# Overview
Describe your project's capabilities here. This acts as the fallback description if the registry needs one.
```
